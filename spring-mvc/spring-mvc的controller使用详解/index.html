请求窄化

同一个项目中，会存在多个接口，例如订单相关的接口都是 /order/xxx 格式的，用户相关的接口都是 /user/xxx 格式的。为了方便处理，这里的前缀（就是 /order、/user）可以统一在 Controller 上面处理。

当类上加了 @RequestMapping("/user") 注解之后，此时，要想访问到 hello ，地址就应该是 /user/hello 或者 /user/hello2
